# 구글 시트 설정 가이드 (SHEET_SETUP)

이 페이지는 **구글 스프레드시트를 데이터 저장소로** 씁니다.
총무는 구글 시트만 수정하면 되고, 페이지는 그 시트의 **웹게시 CSV**를 읽어 자동으로 갱신됩니다.
(재배포 불필요. 단, 웹게시 CSV는 **최대 5분 캐시 지연**이 있습니다.)

> ⚠️ **열 이름과 값 표기는 코드가 그대로 키로 읽습니다.**
> 아래 열 이름(`이름`, `숙박여부`, `구분` 등)과 값(`O`/`X`, `계비`/`대납`, `완료`)을 **바꾸지 마세요.**

---

## 1. 시트 구조 — 탭 2개

스프레드시트 하나에 아래 **탭 2개**를 만듭니다. 탭 이름은 자유지만 아래를 권장합니다.

### 탭 ①  `명단`

| 이름 | 참석여부 | 숙박여부 |
|------|:------:|:------:|
| 강명훈 | O | X |
| 강병훈 | O | O |
| 김병화 | O | O |
| 김외국 | O | X |
| 류정훈 | O | O |
| 홍길동 | X |  |
| … | … | … |

- **1행은 반드시 헤더**: `이름`, `참석여부`, `숙박여부`
- `참석여부` 값은 `O`(참석) 또는 `X`(불참). **불참자도 이 표에 넣고 `참석여부`에 `X`** 를 적으면 "불참인원"으로 집계됩니다. (열이 아예 없거나 비어 있으면 전원 참석으로 간주)
- `숙박여부` 값은 `O`(숙박) 또는 `X`(당일 복귀). **참석자에게만 적용**됩니다. 불참자는 비워도 됩니다.
- 정렬 순서(가나다순 등)는 시트에 넣은 순서 그대로 페이지에 나옵니다.
- 참석/불참/숙박/당일복귀 숫자는 **페이지가 자동 계산**합니다. 따로 적지 마세요.
  - 참석 = `참석여부 O` / 불참 = `참석여부 X` / 숙박 = `참석 O + 숙박 O` / 당일복귀 = `참석 O + 숙박 X`

### 탭 ②  `지출`

| 일자 | 구분 | 항목 | 금액 | 결제자 | 비고 | 상태 |
|------|------|------|-----:|--------|------|------|
| 2026-08-22 | 대납 | 숙소비용 (킴스테이 1박) | 516000 | 정연재 | Airbnb 선결제 | |

- **1행은 반드시 헤더**: `일자`, `구분`, `항목`, `금액`, `결제자`, `비고`, `상태`
- `구분` 값은 **딱 두 가지만**:
  - `계비` = 회비 통장에서 직접 결제
  - `대납` = 개인이 먼저 결제 → 나중에 계비에서 환급
- `금액`은 **숫자만** (예: `516000`). 콤마/원 넣어도 되지만 숫자만 권장. 페이지가 `516,000원`으로 표시합니다.
- `상태` 열: 환급이 끝났으면 `완료`를 적으면 "환급 완료" 뱃지가 뜹니다. 비워두면 "환급 대기".
- **행은 얼마든지 추가 가능**합니다. 회식비/술값/액티비티 등을 계속 추가하세요.

> 회비 납부 여부는 이 페이지에서 다루지 않습니다. 관련 열을 만들 필요 없습니다.

정산 요약은 모두 자동 계산됩니다:
- 계비 직접결제 합 = `구분`이 `계비`인 행의 금액 합
- 개인 대납 합 = `구분`이 `대납`인 행의 금액 합
- 계비 총 지출 = 위 둘의 합
- 대납 환급 대상 = `대납` 행을 **결제자별로 합산**한 목록

## 2. 웹에 게시 (CSV 공개)

1. 구글 시트 상단 메뉴 **파일 → 공유 → 웹에 게시** (`File → Share → Publish to web`)
2. **게시** 대화상자에서:
   - 왼쪽 드롭다운: **`명단`** 탭 선택 (전체 문서 아님)
   - 오른쪽 드롭다운: **쉼표로 구분된 값(.csv)** 선택
3. **게시** 버튼 클릭 → 생성된 URL 복사 → 이게 `명단`의 CSV URL
4. 같은 방법으로 드롭다운을 **`지출`** 탭으로 바꿔 한 번 더 게시 → `지출`의 CSV URL

게시된 CSV URL은 대략 이런 형태입니다:

```
https://docs.google.com/spreadsheets/d/e/2PACX-XXXXXXXX/pub?gid=0&single=true&output=csv
```

---

## 3. 탭별 URL 구분 — `gid` 파라미터

URL 끝의 **`gid=` 값이 탭(시트) 식별자**입니다.

- 웹게시 대화상자에서 **탭을 바꿔가며 각각 게시**하면 `gid`가 다른 두 URL을 얻습니다.
- 첫 번째 탭은 보통 `gid=0`, 두 번째 탭은 다른 숫자(예: `gid=123456789`)입니다.
- 탭의 `gid`는 시트에서 해당 탭을 열었을 때 브라우저 주소창의 `#gid=...` 에서도 확인할 수 있습니다.

예시:

```
명단 탭:  ...&gid=0&single=true&output=csv
지출 탭:  ...&gid=987654321&single=true&output=csv
```

> `output=csv` 가 반드시 포함돼야 CSV로 내려옵니다. `single=true` 는 단일 탭만 내보낸다는 의미입니다.

---

## 4. 페이지에 연결

`index.html` 상단 `CONFIG` 에 위 두 URL을 넣습니다:

```js
const CONFIG = {
  ROSTER_CSV_URL:  "명단 탭 CSV URL",
  EXPENSE_CSV_URL: "지출 탭 CSV URL",
  ...
};
```

- URL을 비워두면 페이지는 `index.html` 안의 **기본(fallback) 데이터**로 표시됩니다.
- fetch에 실패해도 조용히 fallback으로 표시되므로 화면이 깨지지 않습니다.

## 5. 자주 겪는 문제

| 증상 | 원인/해결 |
|------|-----------|
| 시트 수정이 바로 안 보임 | 웹게시 CSV는 **최대 5분 캐시** 지연. 잠시 후 새로고침. |
| 값이 이상하게 잘림 | 항목/비고에 쉼표가 있어도 됩니다(파서가 처리). 단 헤더 이름은 바꾸지 마세요. |
| 금액이 0으로 표시 | `금액` 칸에 숫자가 아닌 문자가 섞였는지 확인 (숫자만 권장). |
| 환급 완료 뱃지가 안 뜸 | `상태` 칸에 정확히 `완료` 라고 적었는지 확인. |
| 정산 섹션이 안 보임 | 의도된 동작. `index.html`의 `SHOW_SETTLEMENT`를 `true`로 바꾸세요(모임 후). |
